# Experimental Assistant v1.0.0 Release Notes

Experimental Assistant v1.0.0 is the first stable public release of the feature-frozen v0.9.9 contract. The public surface remains the `experimental-assistant` distribution, `ea` CLI, and single `$ea` Codex Skill.

## Stable release changes

- Fixed audit-mode routing for `references validate-report`, `export verify-bundle`, and `export verify-archive`; all three remain byte-stable and read-only.
- Added `EA-MODE-COMMAND-BLOCKED` so interaction-mode policy is not reported as an operating-system permission failure.
- Aligned package, CLI, Skill, citation, onboarding, examples, release tooling, and current public documentation on `1.0.0` / `v1.0.0`.
- Added current v1 release-package and detached-signature types while retaining declared v0.9.7–v0.9.9 verification readers.
- Promoted only the public-benchmark-bounded Raman preprocessing, peak-screening, review, and traceable-report path to the stable core.
- Revalidated clean install, v0.9.9→v1 update, rollback, uninstall, project compatibility, reproducibility, SBOM, vulnerability, package, and downloaded-asset contracts for the final candidate.

## Evidence policy

Automated tests, public benchmarks, deterministic Mock integrations, simulated-agent journeys/reviews, and manual artifact inspection are labeled by evidence type. They are not represented as real-user trials, independent expert approval, or live account/service validation.

Historical v0.9.x release records, benchmarks, schemas, protocols, project records, and provenance identifiers are retained without mechanical rewriting.
