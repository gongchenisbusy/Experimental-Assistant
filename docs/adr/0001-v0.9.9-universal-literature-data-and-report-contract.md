# ADR 0001: v0.9.9 universal literature data and report contract

- Status: accepted for implementation
- Date: 2026-07-16
- Baseline: `v0.9.8` (`d8f849baa021a6b1e4c76ee0d8f8624bd5553b51`)
- Target: `v0.9.9`

## Context

The v0.9.8 literature evidence workflow treats six electrical property kinds as a support allowlist. This prevents a user-defined field from entering the same plan, extraction, review, validation, plot, and export path. The v0.9.8 report path also localizes a small set of complete English strings, appends a report-level source-data section, and renders report-bound PNG footers with a fixed bitmap font. Those contracts do not satisfy issue #14 or the approved v0.9.9 plan.

## Decision

### Literature data

Use a versioned `literature-data-schema` as the sole support contract. A schema defines fields, aliases, data types, units and explicit conversions, conditions, missing-value policy, comparability, deduplication, conflict handling, evidence requirements, output behavior, and plotting policy. The six electrical properties become built-in presets loaded through the same validator and engine as project or user schemas.

Planning is zero-write until explicit confirmation. The preview records the canonical schema and semantic hash. Confirmation may persist the dataset only when the same semantic hash is confirmed. A changed hash marks existing extraction state stale and requires an explicit migration or re-plan.

Keep the v0.9.8 record keys and paths readable. New records add schema and field-value structures without rewriting historical datasets. Accepted or edited records remain the only records eligible for downstream statistics, plotting, reports, exports, or durable memory.

### Dynamic report language

Represent generated interpretation, warning, recommendation, caption, and source-data purpose text with stable message keys and structured parameters. Render through a locale catalog. Legacy free text remains readable, but a Chinese report must use a safe localized fallback instead of emitting an unrecognized English sentence.

The semantic contract remains language-neutral and preserves values, units, confidence enums, evidence refs, warning codes, and citations.

### Figure component and footer

Render each figure, caption, concise source-data links, and figure warnings as one figure component in Markdown and HTML. Do not append a standalone `图下数据` or `Figure Source Data` report section. Preserve full roles, columns, hashes, and paths in sidecars and bundles.

Generate new report-bound PNG renderings from the immutable footer-free base. Use a scalable DejaVu Sans font from the packaged Matplotlib dependency, responsive wrapping, high contrast, and a recorded HTML display-scale contract of at least 12 CSS px. Do not overwrite historical renderings.

### Public identity and evidence

Package and expose only `$ea`. Preserve historical project identifiers and readers. Release evidence distinguishes `automated_test`, `simulated_agent`, `mock_fixture`, `public_benchmark`, and `manual_artifact_review`; simulated evidence is never described as real-user or independent-expert validation.

## Consequences

- The CLI gains schema template and validation paths while preserving existing electrical commands.
- Extraction code becomes type- and schema-driven rather than property-allowlist-driven.
- Final-artifact tests, not metadata-only checks, become release blockers.
- A v1.0 promotion can reuse the v0.9.9 feature set; functional fixes return to the v0.9.9 RC loop.
