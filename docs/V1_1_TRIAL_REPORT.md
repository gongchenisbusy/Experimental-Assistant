# Experimental Assistant v1.1.0 Candidate Trial Report

## Scenarios

The candidate is exercised against issue-derived scenarios rather than only unit-level helpers:

1. Protected Raman/PL/AFM-style import into a writable project, including injected provenance failure and rollback.
2. Consult/audit method inspection with write commands still blocked.
3. LabRAM-style PL preview and normalized-versus-raw plot scaling.
4. Six furnace runs, selected sample, method coverage, stage-aware memory, reviewed composite report, HTML delivery, and draft preview.
5. Six-paper fixed literature scope, interruption/resume, Zotero choice, privacy-safe institution state, local PDF ingest, invalid PDF/DOI rejection, and cross-file reconciliation.
6. EA-feedback local collection using active install evidence without GitHub readiness probes.

## Candidate evidence

- Clean v1.0 baseline: 458 tests passed before implementation.
- Foundation adjacent regression: 112 tests passed.
- Literature workflow/pipeline/data-extraction/benchmark regression: 72 tests passed.
- EA-feedback companion: 12 tests passed plus Skill validation and real local-only collector replay.
- Final full suite, public release smoke, native CI, package, signature, SBOM, vulnerability, lifecycle, artifact smoke, and reproducibility results are recorded in `docs/V1_1_RELEASE_DOSSIER.yml` after the release commit is built.

## Safety review

No test or implementation weakens raw-data protection, review requirements, provenance, legal-access boundaries, or private credential handling. The pre-existing local demo workspace is excluded from staging and public release inputs.
