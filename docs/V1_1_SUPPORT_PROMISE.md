# Experimental Assistant v1.1.0 Support Promise

- Maintain the v1 project format and public `$ea` identity across the v1.x line unless a documented migration is required.
- Treat protected raw data, provenance, review binding, export integrity, and public-user privacy regressions as release-blocking defects.
- Support Python 3.11–3.13 on the native CI matrix documented in `docs/SUPPORT_POLICY.md`.
- Keep Zotero, browser, institution access, diagnostics submission, and feedback submission optional and user-authorized.
- Publish exact package checksums, release-package verification metadata, and detached signature evidence for release assets.
- Preserve historical release records and provide documented update, rollback, uninstall, and project verification paths.
